# Python Automations

Collection of Python automations for data and productivity workflows: ETL, data quality, report generation, file organization, backups and API collection. Each script is self-contained, with a CLI (`argparse`), logging and error handling — ready to run directly or schedule via cron / Task Scheduler.

## Scripts

| Script | What it does | Example usage |
|---|---|---|
| `etl_csv_para_sqlite.py` | ETL pipeline: reads CSV, normalizes columns to snake_case, trims whitespace, removes duplicates, adds an audit column and loads into SQLite with volume validation | `python etl_csv_para_sqlite.py --input train.csv --db titanic.db --tabela passageiros` |
| `validador_qualidade_dados.py` | Markdown data-quality (profiling) report: nulls, duplicates, constant columns, high cardinality and outliers (IQR) | `python validador_qualidade_dados.py --input train.csv --output relatorio.md` |
| `gerador_relatorio_excel.py` | Converts a sales CSV into a formatted .xlsx report: formula-driven KPIs, supporting tables and bar/line charts | `python gerador_relatorio_excel.py --input vendas.csv --output relatorio.xlsx` |
| `organizador_arquivos.py` | Organizes a folder into subfolders by file-extension category or by date (YYYY-MM), with `--dry-run` mode and collision handling | `python organizador_arquivos.py --pasta ~/Downloads --dry-run` |
| `backup_automatico.py` | Timestamped .zip backup with integrity check (CRC) and rotation of the N most recent | `python backup_automatico.py --origem ./projetos --destino ./backups --manter 5` |
| `coletor_github_api.py` | Records historical snapshots of GitHub repository metrics (stars, forks, issues...) into a CSV for time-series tracking | `python coletor_github_api.py --repos pandas-dev/pandas --saida historico.csv` |

## Installation

```bash
git clone https://github.com/kagenash/automacoes-python.git
cd automacoes-python
pip install -r requirements.txt
```

Requires Python 3.10+.

## Tests

Core ETL and data-quality functions have unit tests:

```bash
pytest -v
```

## Sample output

The [`exemplos/`](exemplos/) folder contains real output generated by the scripts: a data-quality report for the Titanic dataset (`relatorio_qualidade_titanic.md`), a fictitious sales CSV (`exemplo_vendas.csv`) and the formatted Excel report generated from it (`relatorio_vendas.xlsx`).
